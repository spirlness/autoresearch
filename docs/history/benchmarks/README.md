# Benchmarks

This folder keeps committed historical benchmark evidence.

- `archive/` stores versioned benchmark snapshots that are useful project history.
- `generated/` stores preserved generated outputs that were committed because they support the current written findings.

Useful summaries:

- [`../../findings/benchmark-summary-2026-03-15.md`](../../findings/benchmark-summary-2026-03-15.md) is the detailed benchmark table and experiment recap for the current tuning round.
- [`../../findings/benchmark-findings.md`](../../findings/benchmark-findings.md) is the short list of durable conclusions worth carrying forward when revisiting performance work.

For new local ad-hoc benchmark or probe output, use `results/` instead of this history folder.
